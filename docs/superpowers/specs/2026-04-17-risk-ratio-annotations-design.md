# DFund 风险调整比率注释功能设计

- 日期：2026-04-17
- 页面：`docs/index.html`
- 范围：为右侧“关键业绩指标”中的 `Sharpe Ratio`、`Sortino Ratio`、`Calmar Ratio` 增加轻量注释功能

## 目标

让页面阅读者在不离开当前视线区域的情况下，快速理解三个风险调整比率的含义。

本次设计只解决“快速读懂”问题，不扩展新的业绩指标，不改数据管道，不引入外部依赖。

## 非目标

- 不新增 `Information Ratio`、`Treynor Ratio`
- 不修改 Python 数据生成逻辑
- 不引入第三方 tooltip 库
- 不增加单独说明页或长篇教学内容

## 设计决策

### 1. 入口位置

仅在右侧“关键业绩指标”区域的三个现有比率项中增加说明入口：

- `Sharpe Ratio`
- `Sortino Ratio`
- `Calmar Ratio`

每个指标名称后追加一个小圆形 info 图标，作为说明入口。

### 2. 交互方式

采用方案 A，也就是逐项悬停提示：

- 桌面端：鼠标悬停 info 图标时显示浮层，移出后关闭
- 移动端：点击 info 图标显示浮层，点击页面空白处或再次点击时关闭
- 键盘可访问：图标可聚焦，聚焦后可触发显示，按 `Esc` 关闭
- 任一时刻只显示一个浮层，避免右侧栏出现多个说明同时展开

## 布局与样式

### 1. 视觉原则

沿用当前页面风格：白底、浅边框、小字、轻阴影，不引入重弹窗或深色气泡。

### 2. 浮层位置

默认优先显示在 info 图标右侧；若空间不足，则显示在下方。

浮层应控制为短文本块，避免遮挡整列指标内容。

### 3. 对现有布局的约束

- 不改变右侧栏整体结构
- 不明显拉高单个 `stat-item`
- 不影响指标数值对齐和现有间距
- 移动端下浮层应保持可读，不超出视口

## 文案结构

每个浮层统一采用以下结构：

1. 标题：指标名
2. 白话解释：说明它衡量什么，通常如何理解数值高低
3. 简化公式：帮助读者建立直觉，不追求学术排版

### 文案草案

#### Sharpe Ratio

白话：每承受 1 单位总波动，基金大致换来了多少超额回报，通常越高越好。

公式：`(年化收益率 - 无风险利率) / 年化波动率`

#### Sortino Ratio

白话：和夏普类似，但只惩罚下跌波动，更适合看“亏损风险换来了多少回报”，通常越高越好。

公式：`(年化收益率 - 无风险利率) / 下行波动率`

#### Calmar Ratio

白话：用收益去对比最大回撤，表示“承担一次大回撤，换来了多少年化收益”，通常越高越好。

公式：`年化收益率 / 最大回撤`

## 实现方案

### 1. 数据组织

在 `docs/index.html` 的前端脚本中增加一个本地配置对象，用于集中管理三个比率的说明文案。

建议结构：

```js
const ratioAnnotations = {
  sharpe: {
    title: 'Sharpe Ratio',
    summary: '...白话解释...',
    formula: '(年化收益率 - 无风险利率) / 年化波动率'
  },
  sortino: {
    title: 'Sortino Ratio',
    summary: '...白话解释...',
    formula: '(年化收益率 - 无风险利率) / 下行波动率'
  },
  calmar: {
    title: 'Calmar Ratio',
    summary: '...白话解释...',
    formula: '年化收益率 / 最大回撤'
  }
};
```

这样后续若新增别的比率，可以沿用同一渲染方式。

### 2. 渲染方式

当前 `statList` 由模板字符串一次性渲染。实现时保持这一模式，在三个目标项的名称区域中插入：

- 指标文本
- info 图标按钮
- 对应的说明浮层容器

其余指标保持不变。

### 3. 事件处理

需要补充以下行为：

- `mouseenter` / `mouseleave` 控制桌面端悬停显示
- `click` 控制移动端和键盘用户显示
- `focus` / `blur` 支持键盘浏览
- `Escape` 关闭当前浮层
- 点击页面其他区域时关闭当前浮层

### 4. 无依赖实现

全部使用原生 HTML、CSS、JavaScript 完成，不新增库，不拆文件。

## 可访问性要求

- info 图标使用可聚焦元素，如 `button`
- 为图标提供清晰的 `aria-label`
- 浮层与触发器之间建立可理解的关联关系
- 键盘用户可打开和关闭说明，不依赖鼠标悬停

## 验收标准

### 桌面端

- 三个比率旁都显示统一风格的 info 图标
- 悬停图标时能看到对应说明
- 浮层内容正确，不串项
- 移出后正常关闭
- 不影响现有指标数值排版

### 移动端

- 点击图标可以查看说明
- 点击空白处或再次点击后关闭
- 浮层不超出视口，不遮挡大部分内容

### 回归要求

- 右侧“关键业绩指标”其他条目表现不变
- 页面现有图表、切换器、表格交互不受影响
- 不修改任何数据输出结果

## 实施边界

本次改动仅限 `docs/index.html`。

不新增数据文件，不改 `scripts/update_data.py`，不改部署流程。
